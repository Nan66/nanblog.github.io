初衷

作为一个程序员，没有一个自己的博客感觉是不完整的。之前的读书笔记，专业知识都是通过本地笔记的方式进行记录，现在打算建立一个自己的博客来记录自己的学习过程，同时也用来记录自己的生活。

为什么选择hugo

首先Hugo使用go语言进行开发，速度更快。自己建立这个博客是想要长期维护的，所以随着后面文章越来越多，可能速度是一个比较重要的点。另外一个原因是因为自己目前正在学习go，所以觉得选择hugo来建立自己的博客还可以帮助学习go，何乐而不为呢。

mac上安装hugo

  1. 通过brew进行安装

    Brew 是维护起来最简单轻便的方法。缺点也不严重。默认的安装包将会是 最近的发布版本，因此在下一个版本发布前，将无法提供缺陷修复的版本（除非 你使用 --HEAD 选项来安装）。brew 上的发行版可能会滞后几天，因为它需要 和其他组配合。尽管如此，如果你希望使用一个稳健、被广泛使用的源码，我还是推荐 使用 brew。它不仅有效，而且非常容易升级。

    1. 安装brew

      访问官网查看安装方法，也可看我的博客《homebrew下载》

  2. 通过tarbar进行安装

    下载 tarball 并安装也非常的简单。你需要有少量的命令行技能。 升级也很容易。你仅需要在新的二进制上重复相同的步骤就行。这可以赋予你一定的灵活性，在电脑上 拥有不同的版本。如果你想使用 brew， 那么选择使用二进制安装是个好主意。

  3. 首次运行之前，我们还需要为hugo添加主题模版，因为默认是没有主题的。

    我们可以到https://themes.gohugo.io/ 这里来找到自己喜欢的主题，然后在hugoblog目录下通过git下载喜欢的主题到themes中，然后修改config.toml来应用该主题。

    • 这里我选择的是dream主题
    1
2

    $ cd themes
$ git clone https://github.com/g1eny0ung/hugo-theme-dream.git dream
    • 修改config.toml,添加theme = "dream" and save.

    还有很多可配置的东西在等待我们探索。这里暂不多说。

config.toml是配置文件，一些配置在这里使用

content里存放的是我们的.md 文件也就是文章

static 里存放的是js等一些静态文件

themes是我们的主题文件，默认hugo是没有主题的，所以如果我们装完hugo，没下载主题，运行后就会空白。

其他目录还没有研究明白。

使用

在我们site建好后，即可建立我们的博客了。

1
2

$ hugo new test/test.md
/Users/nan/blog/nanblog/content/test/test.md created

这样会自动创建test目录和test.md文件。直接在相应目录下建md文件也可以。使用hugo命令不是必须的。

但是相关的属性需要自己添加（title，data啥的）

这里需要注意的是一定要在site目录下执行这个命令，其他目录下会报错，无法新建。看网上说要在content/post下写文章，目前感觉都可以。

最后需要注意的是博客的分类什么的需要在md文件前进行标注，而不是通过配置来实现。目前的理解是这样。
