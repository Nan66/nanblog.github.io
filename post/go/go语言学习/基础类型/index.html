定义变量

使用 var 关键字是 Go 最基本的定义变量方式，与 C 语言不同的是 Go 把变量类型放在变量名后面:

//定义一个名称为“variableName”，类型为"type"的变量
var variableName type

//定义三个类型都是“type”的三个变量
var vname1, vname2, vname3 type

//初始化“variableName”的变量为“value”值，类型是“type” 
var variableName type = value

/*
定义三个类型都是"type"的三个变量,并且它们分别初始化相应的值
vname1 为 v1，vname2 为 v2，vname3 为 v3 
*/
var vname1, vname2, vname3 type= v1, v2, v3

简单写法：

可以直接忽略类型声明

1
2
3
4

/*
定义三个变量，它们分别初始化相应的值 vname1 为 v1，vname2 为 v2，vname3 为 v3 然后 Go 会根据其相应值的类型来帮你初始化它们
*/
var vname1, vname2, vname3 = v1, v2, v3

（为什么go可以忽略类型，自动匹配吗，go是强类型还是弱类型）

继续简化，:=这个符号直接取代了 var 和 type,这种形式叫做简短声明。 不过它有一个限制，那就是它只能用在函数内部;在函数外部使用则会无法编译通过，所 以一般用 var 方式来定义全局变量。

4

/*
定义三个变量，它们分别初始化相应的值 vname1 为 v1，vname2 为 v2，vname3 为 v3 编译器会根据初始化的值自动推导出相应的类型
*/
vname1, vname2, vname3 := v1, v2, v3

整型

整数类型有无符号和带符号两种。Go 同时支持 int 和 uint，这两种类型的长度相同，但具体 长度取决于不同编译器的实现。当前的 gcc 和 gccgo 编译器在 32 位和 64 位平台上都使用 32 位来表示 int 和 uint，但未来在 64 位平台上可能增加到 64 位。Go 里面也有直接定义好位数的类型:rune, int8, int16, int32, int64 和 byte, uint8, uint16, uint32, uint64。其中 rune 是 int32 的别称，byte 是 uint8 的别称。

需要注意的一点是，这些类型的变量之间不允许互相赋值或操作，不然会在编译时引起编 译器报错。

如下的代码会产生错误

1
2
3
4

var a int8 
var b int32 
c:=a + b
//另外，尽管 int 的长度是 32 bit, 但 int 与 int32 并不可以互用。

浮点数类型

浮点数的类型有 float32 和 float64 两种(没有 float 类型)，默认是 float64。

这就是全部吗?No!Go 还支持复数。它的默认类型是 complex128(64 位实数+64 位虚 数)。如果需要小一些的，也有 complex64(32 位实数+32 位虚数)。复数的形式为 RE + IMi，其中 RE 是实数部分，IM 是虚数部分，而最后的 i 是虚数单位。下面是一个使用复数的 例子:

1
2
3

var c complex64 = 5+5i
//output: (5+5i)
fmt.Printf("Value is: %v", c)

字符串

  • 默认值是空字符串 “"。
  • ⽤用索引号访问某字节，如 s[i]。
  • 不能⽤用序号获取字节元素指针，&s[i] ⾮非法。 • 不可变类型，⽆无法修改字节数组。
  • 字节数组尾部不包含 NULL。

Go 中的字符串都是采用 UTF-8 字符集编码。字符串是用一对双引号

("")或反引号( )括起来定义，它的类型是 string。

no, yes, maybe := "no", "yes", "maybe" // 简短声明，同时声明多个变量 japaneseHello := "Ohaiou" // 同上

Go中，字符串是不可变的（原因应该是和java一样，更安全）

例如：下面的代码编译时会报错

1

s[0] = 'c'

但如果真的想要修改怎么办呢?下面的代码可以实现:

1
2

c := []byte(s) // 将字符串 s 转换为 []byte 类型 
c[0] = 'c'
s2 := string(c) // 再转换回 string 类型
fmt.Printf("%s\n", s2)

Go中可以使用+操作符来连接两个字符串

错误类型

Go 内置有一个 error 类型，专门用来处理错误信息，Go 的 package 里面还专门有一个包

errors 来处理错误:

1
2
3
4

err := errors.New("emit macho dwarf: elf header corrupted")
if err != nil { 
	fmt.Print(err)
}

Go语言中不支持隐式类型转换，由窄转宽也不可以。

不能将其他类型当作bool类型来使用

4

a := 100

    println("true")
}

注⚠️

​ GO语言中的变量要被使用到，不能存在没使用到的类型，如果有，会报错。

​ import的包同样的道理，也要使用到。

总而言之，就是不能存在没有用的东西，目的可能是为了防止代码冗余（比如定义了一些没有用到的变量，浪费地方，观赏性差）

这里的使用是要用到，比如定义一个变量a，然后把它打印，这种情况就是使用到了。

而像对a赋值这种操作并不是使用到了a。
