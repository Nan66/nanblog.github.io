GoROOT、GOPATH、GOBIN 的理解

GOROOT:Go语言安装根目录的路径，也就是GO语言的安装路径

GOPATH: 若干工作区目录的路径，即自定义的工作空间

GOBIN: GO程序生成的可执行文件（executable file）的路径

设置GOPATH的意义

GOPATH:Go语言的工作目录，值是一个目录的路径，也可以是多个目录的路径，每个目录都代表GO语言的一个工作区(Workspace)

工作区的作用：

  1. 放置GO语言的源码文件，
  2. 安装后的归档文件（archive file， 即 .a为扩展名的文件）
  3. 可执行文件（executable file）

什么是归档文件

当我们在编译时，会依赖很多库，你可以理解为.a文件就是那些库的编译中间结果，有了那些，只要你不做改动，下次编译的时候就直接拿过来了，不需要重复编译了。

Go项目的生命周期

  1. 编码
  2. 依赖管理
  3. 构建
  4. 测试
  5. 安装

基本围绕Gopath和工作区进行，所以，背后至少有3个点：

  1. GO语言源码的组织方式

    1. GO语言以代码包为基本单位

    2. 一个代码包中可包含多个.go的源码文件，这些源码文件都需被声明属于同一个代码包

    3. 代码包的名称一般会与源码文件所在的目录同名，如果不同名，则在构建、安装的过程中会以代码包名称为准

    4. 每个代码包都会有导入路径

    5. 工作区中，一个代码包的导入路径实际上就是src字母到该报的实际存储位置的相对路径

    6. 总结

      1. Go语言源码的组织方式是以GOPATH、工作区、src目录和代码为主线的
      2. 通常，GO语言的源码文件都需存放在GOPATH包含的某个工作区（目录）中的src目录下的某个代码包中
      3. import “github.com/labstack/echo” //导入代码包

  3. 了解源码安装后的结果

    1. 源码文件通常会被放在某个工作区的src子目录下

    2. 归档文件（.a）就会放在工作区的pkg子目录

    3. 可执行文件，放进工作区的bin子目录

    4. 代码包导入路径 github.com/labstack/echo

    5. 归档文件的相对目录与pkg目录之间还有一级目录，叫做平台相关目录

      1. 平台相关目录：build（“构建”）的目标操作系统、下划线、目标计算架构的代号组成的

  5. 是否理解构建和安装Go程序的过程

    1. 构建使用命令go build，安装使用命令go install
    2. 构建和安装代码包的时候都会执行编译、打包等操作，并且，这些操作生成的任何文件都会先被保存到某个临时的目录中。
    3. 构建的是库源码文件，那么操作后产生的结果文件只会存在于临时目录中。这里的构建的主要意义在于检查和验证。
    4. 构建的是命令源码文件，那么操作的结果文件会被搬运到源码文件所在的目录中

  7. go build的用途和用法

    1. go build命令的时候，默认不会编译目标代码包所依赖的那些代码包

    2. 被依赖的代码包的归档文件不存在，或者源码文件有了变化，那它还是会被编译。

    3. 强制编译-a,目标代码包、依赖的代码包总会被编译

    4. 如何确定哪些代码包被编译了？

      1. go build -x //具体执行了哪些操作 -n //查看不执行
      2. go build -v //go build命令编译的代码包的名称
