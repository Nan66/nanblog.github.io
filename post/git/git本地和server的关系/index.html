git本地和server的关系

2021-01-10
git
281 words 1 min read

文章目录

    • 开发流程
    • 原因

开发流程

download server的代码后，之后开发时，在开发前都要新建一个分支，然后在分支上进行开发，开发完成后，将server的master更新到你的分支上，把冲突处理了，然后在讲代码push到的server上进行code review，review后在将分支合入master中。

原因

本地download server的代码后，随着其他人不断上传代码，我们的版本就会落后，所以我们在push代码到server前，要将master的代码更新到我们的分支上，以此来保证我们分支的代码是最新的。

（是add前pull还是commit前pull）- 应该是commit前pull就好了。
